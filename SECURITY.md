# Security Policy

This policy applies to all UptimePage repositories that do not ship their own `SECURITY.md`.

## Reporting a vulnerability

Email **slima4.u8@gmail.com** with subject `[SECURITY]`.

**Please do not** open a public GitHub issue for security vulnerabilities.

## What to include

- Description of the issue
- Steps to reproduce
- Impact assessment
- Whether you've disclosed it elsewhere

## Response timeline

- Acknowledgement: within 48 hours
- Initial assessment: within 7 days
- Fix timeline depends on severity

## Safe harbor

Good-faith security research consistent with this policy will not result in legal action.
