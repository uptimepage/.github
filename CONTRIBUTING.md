# Contributing to UptimePage

Thanks for considering a contribution. This is the org-wide default; individual repositories may ship their own `CONTRIBUTING.md` that takes precedence.

## License agreement

Your contributions are licensed under the same license as the repository you contribute to (see that repo's `LICENSE`).

We use the [Developer Certificate of Origin (DCO)](https://developercertificate.org/) to ensure contributors have the right to submit their work. Every commit must include a `Signed-off-by` line:

    Signed-off-by: Your Name <your.email@example.com>

Add it automatically with `git commit -s`. Configure your git identity once:

    git config --global user.name "Your Name"
    git config --global user.email "your.email@example.com"

PRs without DCO sign-off cannot be merged.

## Code of conduct

Be kind. Be specific. Assume good faith.

## Before you open a PR

1. **Open an issue first** for non-trivial changes — describe what you want to do and why, so you don't build something we can't accept.
2. **Run the repo's tests and linters** and make sure they pass.
3. **Keep the change focused** — one logical change per PR.

## Reporting security issues

Do **not** open a public issue. See [SECURITY.md](SECURITY.md).
